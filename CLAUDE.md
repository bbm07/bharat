## Project Overview

A FastAPI web application for searching Indian (NSE/BSE) and US stock data. The backend uses **yfinance** to fetch live data from Yahoo Finance; the frontend is a single-page HTML/JS UI served inline from the FastAPI app (no separate frontend build step).

## Commands

```bash
# Install dependencies
pip install -r requirements.txt

# Run the dev server (auto-reload)
uvicorn main:app --reload

# Run on a specific port
uvicorn main:app --host 0.0.0.0 --port 8000
```

The app is then available at `http://localhost:8000`.  
API docs (auto-generated by FastAPI): `http://localhost:8000/docs`

## Architecture

Everything lives in `main.py`:

- `GET /` — serves the full HTML/CSS/JS UI as an inline string (no template engine, no static files folder).
- `GET /api/stock?symbol=<ticker>` — looks up a stock and returns JSON.
- `get_ticker(symbol)` — tries appending `.NS` (NSE), then `.BO` (BSE), then no suffix (US stocks) until `yfinance` returns a valid price. Raises `HTTPException(404)` if all attempts fail.

## Key Conventions

- **Indian ticker format**: NSE stocks use `SYMBOL.NS`, BSE use `SYMBOL.BO`. Users type bare symbols (e.g. `RELIANCE`); the suffix is tried automatically.
- **Mock data mode**: The sandbox blocks outbound requests to Yahoo Finance. The current `main.py` uses a `MOCK_STOCKS` dict for demo purposes. When deploying outside the sandbox, replace `fetch_stock()` with a call to `get_ticker()` (the yfinance version is preserved as a comment pattern in the code).
- **No template files**: The UI HTML is returned as a string from the `ui()` route — keep it that way to avoid adding a `templates/` folder dependency.
